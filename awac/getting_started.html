AWAC: Android Web App Container

Getting started with AWAC

If you just want to see what AWAC can do for you, then install AWAC from the Google Play Store. AWAC acts as a container for a specific type of web-app. By default it will run the introductory web-app that is included as part of the install. However you can also use a web-link to launch AWAC to run your own web-app.

Below are links to launch some web-apps hosted on GitHub:

  • acuth.github.io/sample/highline.html
    - a single web-page modified to run in AWAC
  • acuth.github.io/sample/info.html
    - a single web-page modified to run in AWAC
  • acuth.github.io/sample/wizard1.html
    - three page web-app showing a 'wizard-like' interface
  • acuth.github.io/mdl/index.html
    - a single web-page that uses Material Design Lite
  • acuth.github.io/test/index.html
    - the latest version of the introductory web-app

In addition you can use the launch URL to specify a log file on your mobile device to capture and view output from console.log.

  • wizard1.html&log=log.txt
    - 'wizard-like' interface with output directed to log.txt in /Documents/Awac

Customising AWAC

If you want to use AWAC to develop your own Android applications then it is available as an Android Studio project on GitHub. By downloading the project and making some simple configuration changes, you can build your own APK for submission to various app stores.

Examples of simple configuration changes you can make without the need for any coding include:

  • App name, logo + default color scheme
  • Web-app location (either a remote URL or local web assets)
  • App package name
